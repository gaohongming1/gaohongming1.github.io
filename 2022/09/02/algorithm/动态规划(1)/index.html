# 最长回文子串

给你一个字符串 s ，找到 s 中最长的回文子串。

​ 本体有两种解法一种是中心扩展法，另一种就是动态规划解法。动态规划方式可以考虑如何将大问题变为由小问题转换而来。求解字符串 s 的整体的最长回文串可以变为：求解下标 0 到下标 n 的字符串 dp [0][n] 是不是回文串。在考虑整体问题转化为子问题：

  • 当 s [n] ==s [0] dp [i][n] = dp [i+1][n-1]
  • 当 s [n] !=s [0] dp [i][n] = false

# 回文子串

给你一个字符串 s ，请你统计并返回这个字符串中 回文子串 的数目。

回文字符串 是正着读和倒过来读一样的字符串。

子字符串 是字符串中的由连续字符组成的一个序列。

具有不同开始位置或结束位置的子串，即使是由相同的字符组成，也会被视作不同的子串

​ 这个题是针对上面的题的变形，不过变了一下换成了统计数量，核心状态转移方程还是没有变。同样适用于中心扩展法和动态规划方法

# 最长回文子序列

给你一个字符串 s ，找出其中最长的回文子序列，并返回该序列的长度。

子序列定义为：不改变剩余字符顺序的情况下，删除某些字符或者不删除任何字符形成的一个序列。

​ 这个题相比于上面的状态转移方程会有一点变形，我们可以假设 dp [0][n] 代表 0-n 的串经过一系列操作之后可以形成的最大回文串长度，那么 dp [0][n] = dp [1][n-1] 当 s [0]==s [n] 否则 dp [0][n] = max (dp [1][n],dp [1][n-1])

# 最大子数组和

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组 是数组中的一个连续部分。

注意关键点 连续子数组 这个也就代表后边的解和前面的解是有关系的，可以采用贪心的思想，每次加上当前元素的目的都是向得到比当前元素单个更大的元素，否则还不如从当前元素开始呢。

# 乘积最大子数组

给你一个整数数组 nums ，请你找出数组中乘积最大的非空连续子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。测试用例的答案是一个 32 - 位 整数。子数组 是数组的连续子序列。

这个题和上边那个很相似，唯一的区别就是加法换成了乘法，所以这里需要注意，负负得正的情况，所以需要维护当前元素可以获得的最小值.

# 数组找规律系列

# 爬楼梯

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

可以想我们有几种办法登上第 N 个台阶？可以从第 n-2 跳上也可以从 n-1 跳上。

所以 dp [n] = dp [n-1] +dp [n-2]

# 买卖股票的最佳时机

给定一个数组 prices ，它的第 i 个元素 prices [i] 表示一支给定股票第 i 天的价格。你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

这个题可以用动态规划，参考买卖股票系列动态规划，也可以用贪心算法。贪心的策略是在每一天我们假设已经以之前的某一天的最低价格买进了股票，算下当天卖出的最大利润，如果当天价格小于历史最低价格那就更新历史最低价格

# 跳跃游戏

给定一个非负整数数组 nums ，你最初位于数组的 第一个下标 。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标。

这个同样可以采用贪心的策略，还有点动态规划的思想。通过不断的更新可以跳跃到的最大距离，当最大的跳跃距离超过数组的长度的时候就可以判定可以跳到最后一个下标。

# 跳跃游戏 II

给你一个非负整数数组 nums ，你最初位于数组的第一个位置。数组中的每个元素代表你在该位置可以跳跃的最大长度。你的目标是使用最少的跳跃次数到达数组的最后一个位置。假设你总是可以到达数组的最后一个位置。

同样可以使用贪心的策略维护最大可跳跃距离，当跳到此窗口的最大可跳跃距离后代表进行了一次跳跃动作 + 1.

这里在讨论下动态规划的思路：设 dp [n] 代表跳转到位置 N 所需的最小步骤。在看下这个数和前面的关系，假设 nums [k] 位置可以跳到 n, 那么 dp [n] 有可能等于 dp [k]

总结：循环数组，并且动态的维护每个元素到可跳转位置的最少次数。
